# PDF documents

Three approaches to generating PDFs, picked by what's already in your stack:

- **Typesetting language** → [[typst]]. New, fast, LaTeX-shaped but ergonomic. Best for from-scratch documents.
- **HTML/CSS via React** → [[react]]. Use when your team already speaks React/TypeScript and your "PDF" is shaped like a webpage.
- **Math symbols cheat sheet** → [[mdmath_symbols]]. LaTeX-flavoured math syntax that works in KaTeX, MathJax, GitHub Markdown, mdbook, Obsidian, and Typst's math mode.

For LaTeX itself there's no separate article in this section yet — the [LaTeX project page](https://www.latex-project.org/) and [Overleaf](https://www.overleaf.com/learn) are the canonical entry points. The [[typst]] article discusses when to pick it over LaTeX.
