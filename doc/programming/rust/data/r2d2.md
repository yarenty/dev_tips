# r2d2 — synchronous connection pool

**Main link:** <https://github.com/sfackler/r2d2>

## Summary

`r2d2` is the canonical **synchronous** generic connection pool for Rust, written by Steven Fackler. It predates async/await and is still the right choice when working with blocking drivers — most notably classic `diesel` (sync API), `rusqlite`, `rust-postgres` (sync), and `redis-rs` in blocking mode. Backend coverage is broad via per-driver adapter crates (`r2d2-postgres`, `r2d2-sqlite`, `r2d2-mysql`, `r2d2-redis`, `r2d2-memcache`, …).

## Insight

Use `r2d2` when your codebase is synchronous (no Tokio runtime). For an async codebase, default to `deadpool`/`bb8` instead. The async pools (`bb8`, `mobc`, `deadpool`) all explicitly list `r2d2` as their API inspiration — if you've used one you've used them all. Gotchas: the `MongoDB` adapter is deprecated because the official `mongodb` driver pools internally; ditto modern `mongodb-rust-driver`. The modern `diesel-async` brings in its own pool — don't double-pool. The `r2d2_postgres` adapter wraps the **sync** `postgres` crate, not `tokio-postgres` — match versions carefully.

`r2d2` is agnostic to the connection type it manages: implement `ManageConnection` to wrap a new backend.

| Backend | Adapter |
|---|---|
| rust-postgres | `r2d2-postgres` |
| redis-rs | enable `r2d2` feature on `redis` |
| rust-memcache | `r2d2-memcache` |
| rust-mysql-simple | `r2d2-mysql` |

Quick example:

```rust
use std::thread;

fn main() {
    let manager = r2d2_foodb::FooConnectionManager::new("localhost:1234");
    let pool = r2d2::Pool::builder().max_size(15).build(manager).unwrap();

    for _ in 0..20 {
        let pool = pool.clone();
        thread::spawn(move || {
            let conn = pool.get().unwrap();
            // use the connection; returned to the pool when it falls out of scope.
        });
    }
}
```
