# Digital Twin

Notes on **digital twins** and the closely-related **cyber-physical systems** (CPS) — virtual representations of physical assets kept continuously synchronised with their real-world counterparts via sensor data. Spans the conceptual definition, the academic CPS framing, vendor tooling (NVIDIA Omniverse, Scilab/Simulink), and a concrete deployment case (BMW iFACTORY).

If you're new to the topic, read in this order:

1. **[Definition & taxonomy](definition.md)** — what a digital twin is and how it differs from a simulation.
2. **[CPS — academic framing](cps.md)** — same idea seen through the academic / Industry 4.0 lens.
3. **[ISC 2022 keynote](isc2022.md)** — the modern industrial-scale narrative (NVIDIA + BMW).
4. **[BMW iFACTORY](bmw.md)** — concrete deployment.
5. **[NVIDIA Omniverse](nvidia_omniverse.md)** — the dominant tooling stack.
6. **[Scilab](scilab.md)** / **[Simulink](simulink.md)** — engineering-tier modelling tools.
7. **[Curated reading list](links.md)** — IBM, MathWorks, Siemens references.
